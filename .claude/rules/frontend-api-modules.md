# Frontend API conventions

The API layer is built on `openapi-fetch` + `openapi-react-query`, typed
against `src/api/generated/schema.ts` (generated by `openapi-typescript`).

## The client

`src/api/client.ts` exports:

- `$api` — the `openapi-react-query` client; use this in all hooks and components
- `apiClient` — the raw `openapi-fetch` client; only needed for one-off calls outside React (e.g. bootstrap)
- `initApiAuth(getter)` — called once by `ProtectedApp` to register the Auth0 token getter
- `ApiRequestError` — error class for typed error handling

## Using the client directly

```ts
import { $api } from '@/api/client'

const { data, isPending } = $api.useQuery('get', '/api/v1/projects')

const mutation = $api.useMutation('post', '/api/v1/projects', {
  onSuccess: () => queryClient.invalidateQueries({ queryKey: ['projects'] }),
})
mutation.mutate({ body: { name: 'My Project' } })
```

## Named hooks (optional but preferred for reuse)

Write a thin hook in `src/api/<domain>/hooks.ts` when:
- the hook is used in more than one place, or
- it needs cache invalidation, `enabled` conditions, or optimistic updates

```ts
// src/api/project/hooks.ts
import { $api } from '../client'
import { useQueryClient } from '@tanstack/react-query'

export const projectKeys = {
  all: () => ['projects'] as const,
  detail: (id: number) => ['projects', id] as const,
}

export function useProjects() {
  return $api.useQuery('get', '/api/v1/projects')
}

export function useCreateProject() {
  const queryClient = useQueryClient()
  return $api.useMutation('post', '/api/v1/projects', {
    onSuccess: () => queryClient.invalidateQueries({ queryKey: projectKeys.all() }),
  })
}
```

No `requests.ts` or `types.ts` files. Types are inferred from the schema.

## Generated files — never edit by hand

| File | How it's generated |
|---|---|
| `src/api/generated/schema.ts` | `npm run openapi:types` (openapi-typescript) |
| `src/api/generated/rbac.gen.ts` | `npm run generate:types` (scripts/generate-types.mjs) |
| `packages/schema/src/generated/constraints.gen.ts` | `npm run generate:types` |

Run `npm run openapi:generate` from `apps/studio-app/` to regenerate everything.

## Step-by-step: adding a new endpoint

1. Run `npm run openapi:generate` to sync the spec and regenerate `schema.ts`.
2. Use the OpenAPI MCP tool (`find_endpoints`, `describe_endpoint`) to confirm the exact path, params, and body shape.
3. Call `$api.useQuery` or `$api.useMutation` inline, or wrap in a named hook if reused.
4. Path strings must match `schema.ts` exactly — copy from the MCP tool output.
