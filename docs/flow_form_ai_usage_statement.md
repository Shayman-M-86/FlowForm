# How I Use AI in the FlowForm Project

## Purpose

This document explains how I use AI while building FlowForm. I wanted to include this because AI tools are part of my workflow, but they are not building the project for me. I still design the architecture, write most of the code myself, and make sure I fully understand how everything works.

For me, AI is more like a **thinking partner and research tool** rather than something that writes the whole project.

---

## My General Approach

I try to use AI in a **minimal and controlled way**.

My goal is always to understand the system I'm building. I don't want to copy‑paste large blocks of code I don't understand. I want FlowForm to actually feel like *my project*.

Most of the structure, design decisions, and architecture come from my own planning. AI mostly helps me think through ideas, explore alternatives, or refine things that I have already started.

---

## Where I Use AI

### Brainstorming and Idea Development

One of the main ways I use AI is for brainstorming.

When I'm planning a feature or trying to design part of the system, I'll often open ChatGPT in the browser and talk through ideas. Sometimes I ask questions like:

- "What are common ways to structure this kind of system?"
- "Is this architecture reasonable?"
- "What are some alternative approaches?"

This helps me explore options, but I still make the final decisions myself.

---

### Research and Documentation

AI is also helpful when I'm trying to understand documentation or industry practices.

Instead of searching through a lot of scattered information, I can ask questions and then go verify the answers in official documentation.

I also go back and forth with AI when writing project documentation. It helps me:

- organize ideas
- structure documents
- refine explanations

Even then, I usually spend time reviewing and editing the documents so they reflect what I actually built.

---

### Coding Assistance

When writing code, I try to write most of it myself.

My main AI assistance during coding is **inline autocomplete** using GitHub Copilot. This helps speed up smaller things like:

- repetitive code
- syntax suggestions
- simple function scaffolding

Sometimes I open ChatGPT to ask about patterns or ways to structure a piece of code, especially when I want to follow a common or well‑established approach.

But I still write and adapt the code myself so I fully understand it.

---

### Light Editing and Refactoring

Occasionally I use AI tools to help with:

- small refactors
- reorganizing code
- rewriting parts of documentation

Even when I do this, I review everything carefully and usually modify the output before committing it to the project.

---

## Tools I Use

These are the AI tools currently part of my workflow:

### ChatGPT (Browser)

Used mainly for:

- brainstorming
- discussing architecture
- researching concepts
- drafting documentation

### OpenAI Codex (VS Code)

Used inside my editor when I want quick suggestions or small edits.

### GitHub Copilot

Used for inline autocomplete while coding.

This mostly helps with small productivity improvements rather than writing full features.

---

## Why I Use AI This Way

My main goal is to **learn and understand the systems I'm building**.

If I relied on AI to generate everything, I would lose that understanding. By keeping AI as a supporting tool instead of the main driver, I can still move faster while making sure the project truly reflects my own work.

FlowForm is something I want to be proud of as a real portfolio project, so I try to stay hands‑on with the design, implementation, and problem solving.

---

## Summary

AI is part of my workflow for the FlowForm project, but it is used mainly as:

- a brainstorming partner
- a research assistant
- a documentation helper
- a light coding aid

Most of the architecture, structure, and implementation decisions are made by me. I try to use AI in a thoughtful way so I stay fully involved in building and understanding the project.

The goal is to build FlowForm in a way that genuinely reflects my own thinking and development process.
